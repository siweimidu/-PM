# 🚀 部署到 GitHub Pages 指南

## 步骤说明

### 1️⃣ 进入仓库设置
- 打开你的仓库：https://github.com/siweimidu/PeriodicTablePM
- 点击顶部的 **Settings**（设置）标签

### 2️⃣ 找到 Pages 设置
- 在左侧边栏中，找到并点击 **Pages**
- 如果没有看到 Pages，可能在 "Code and automation" 分类下

### 3️⃣ 配置 Pages 源
- 在 **Source** 部分，选择 **Deploy from a branch**
- 在 **Branch** 下拉菜单中选择 **main**
- Folder 保持为 **/(root)**
- 点击 **Save** 保存

### 4️⃣ 等待部署
- GitHub 会开始构建你的站点
- 通常需要 1-3 分钟
- 刷新页面后，你会看到顶部显示部署成功的消息
- 并显示你的网站链接：`https://siweimidu.github.io/PeriodicTablePM/`

### 5️⃣ 访问在线版本
- 点击生成的链接即可访问你的在线应用
- 也可以直接访问：https://siweimidu.github.io/PeriodicTablePM/

## 📝 注意事项

- ✅ 首次部署可能需要几分钟时间
- ✅ 每次推送到 main 分支都会自动重新部署
- ✅ 如果看到 404 错误，请等待几分钟再刷新
- ✅ 确保 index.html 文件在仓库根目录

## 🔧 故障排查

### 问题：显示 404 错误
**解决方案：**
1. 确认已在 Settings > Pages 中启用了 GitHub Pages
2. 确认分支设置为 main
3. 等待 2-5 分钟让 GitHub 完成部署
4. 检查 index.html 是否在仓库根目录

### 问题：页面样式加载失败
**解决方案：**
1. 确保所有 CSS 和 JS 文件路径正确
2. 使用相对路径而不是绝对路径
3. 检查浏览器控制台的错误信息

### 问题：图片不显示
**解决方案：**
1. 确认图片文件已上传到仓库
2. 使用正确的相对路径引用图片
3. 检查图片文件名大小写是否匹配
